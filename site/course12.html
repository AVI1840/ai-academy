חלק א׳: מה זה Claude Code?

Claude Code הוא עוזר פיתוח שרץ בטרמינל — הוא קורא את הקוד שלכם, מבין את המבנה, ויכול לכתוב, לתקן ולשפר קוד. כמו מתכנת זוגי חכם שמכיר את כל הפרויקט.

מה Claude Code יכול לעשות

  • לכתוב קוד חדש על בסיס תיאור בשפה טבעית
  • לתקן באגים ולהסביר שגיאות
  • לעשות Refactoring לקוד קיים
  • ליצור טסטים אוטומטיים
  • לכתוב תיעוד טכני
  • לנתח קוד ולמצוא בעיות אבטחה
  • לעבוד עם Git — commits, branches, PRs
  • להריץ פקודות ולנתח תיקיות שלמות

למה זה רלוונטי לממשלה?

  • האצת פיתוח מערכות פנימיות
  • תחזוקת קוד Legacy — הבנה ושיפור של מערכות ישנות
  • אוטומציה של תהליכי DevOps
  • כתיבת סקריפטים לעיבוד נתונים
  • Code Review אוטומטי — מציאת באגים ובעיות אבטחה

חלק ד׳: CLAUDE.md — הגדרת הקשר לפרויקט

קובץ CLAUDE.md בשורש הפרויקט מלמד את Claude Code על הפרויקט שלכם — סטנדרטים, מבנה, וכללים:

# פרויקט: מערכת ניהול תביעות — ביטוח לאומי ## מבנה הפרויקט - src/services/ — לוגיקה עסקית - src/api/ — נקודות קצה - src/models/ — מודלים של בסיס נתונים - tests/ — טסטים ## כללים - כתוב קוד ב-TypeScript - השתמש ב-async/await - כל פונקציה חייבת type hints - טסטים עם Jest - הודעות commit בעברית ## סטנדרטים - אין console.log בקוד production - כל endpoint חייב validation - טיפול בשגיאות עם try/catch

חלק ה׳: Agent Skills — הוראות חוזרות

Skills הם הנחיות שכותבים פעם אחת ו-Claude Code משתמש בהן שוב ושוב:

# skill: write-tests ## description: כתיבת טסטים לפי הסטנדרט שלנו כשאתה כותב טסטים: 1. השתמש ב-Jest 2. כתוב לפחות 3 test cases: happy path, edge case, error case 3. השתמש ב-describe/it בעברית 4. הוסף mock לכל קריאת API חיצונית
# skill: code-review ## description: בדיקת קוד לפי הסטנדרטים שלנו בדוק: - אבטחה: SQL injection, XSS, הרשאות - ביצועים: שאילתות כבדות, לולאות מיותרות - קריאות: שמות משתנים, תיעוד, מבנה - טסטים: האם יש כיסוי מספיק?

חלק ו׳: טיפים מתקדמים

  • היו ספציפיים: "צור endpoint ב-Express שמקבל POST עם שם ומייל" עדיף על "תעשה API"
  • תנו הקשר: "אנחנו משתמשים ב-TypeScript, PostgreSQL, ו-Express"
  • בקשו הסבר: "תתקן ותסביר מה היה הבאג ולמה הפתרון עובד"
  • בדקו תמיד: הריצו טסטים, בדקו קוד, אל תעשו deploy עיוור
  • עבודה מקבילית: הריצו כמה instances על חלקים שונים בו-זמנית
  • שילוב GitHub: "צור PR עם תיאור מפורט" / "בדוק את ה-PR של עמית"
  • MCP Servers: הרחיבו יכולות עם כלים חיצוניים — DB, דפדפן, ועוד
