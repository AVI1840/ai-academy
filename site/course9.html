חלק א׳: מה זה Claude Code?

Claude Code הוא עוזר פיתוח שרץ בטרמינל — הוא קורא את הקוד שלך, מבין את המבנה, ויכול:

  • לכתוב קוד חדש
  • לתקן באגים
  • לעשות Refactoring
  • לכתוב טסטים
  • לעבוד עם Git
  • להריץ פקודות
  • לנתח קבצים ותיקיות שלמות

למה זה רלוונטי לביטוח לאומי?

  • האצת פיתוח מערכות פנימיות
  • תחזוקת קוד Legacy
  • אוטומציה של תהליכי DevOps
  • כתיבת סקריפטים לעיבוד נתונים
  • Code Review אוטומטי

חלק ג׳: 10 שימושים מעשיים לביטוח לאומי

1 תיקון באגים:

> תבדוק למה הפונקציה calculateBenefit מחזירה ערך שגוי כשהגיל מעל 67

2 כתיבת טסטים:

> כתוב unit tests לכל הפונקציות בקובץ eligibility-checker.js

3 Refactoring:

> שפר את הקוד בקובץ claims-processor.py — הפוך אותו לקריא יותר, הוסף type hints ותיעוד

4 סקריפט לעיבוד נתונים:

> כתוב סקריפט Python שקורא קובץ CSV של תביעות, מסנן תביעות פתוחות מעל 90 יום, ומייצר דוח Excel

5 תיעוד קוד:

> הוסף תיעוד מפורט (docstrings) לכל הפונקציות בתיקייה src/services/

6 Code Review:

> בדוק את השינויים ב-Git diff האחרון. חפש: בעיות אבטחה, באגים פוטנציאליים, שיפורי ביצועים

7 המרת קוד:

> המר את הסקריפט הזה מ-VBScript ל-Python. שמור על אותה לוגיקה בדיוק

8 יצירת API:

> צור REST API endpoint שמקבל מספר תביעה ומחזיר סטטוס. השתמש ב-Express.js עם validation

9 אוטומציית Git:

> claude commit # יוצר commit message אוטומטי ומדויק > בדוק את כל ה-PR הפתוחים ותן סיכום של השינויים בכל אחד

10 ניתוח קוד Legacy:

> נתח את הקוד בתיקייה legacy/ והסבר מה כל קובץ עושה. צור מפת תלויות
📄

חלק ד׳: CLAUDE.md — הגדרת הקשר לפרויקט

קובץ CLAUDE.md בשורש הפרויקט מלמד את Claude Code על הפרויקט שלך:

# פרויקט: מערכת ניהול תביעות — ביטוח לאומי ## מבנה הפרויקט - src/services/ — לוגיקה עסקית - src/api/ — נקודות קצה - src/models/ — מודלים של בסיס נתונים - tests/ — טסטים ## כללים - כתוב קוד ב-TypeScript - השתמש ב-async/await - כל פונקציה חייבת type hints - טסטים עם Jest - הודעות commit בעברית ## סטנדרטים - אין console.log בקוד production - כל endpoint חייב validation - טיפול בשגיאות עם try/catch
🧩

חלק ה׳: Agent Skills — הוראות חוזרות

Skills הם הנחיות שאתה כותב פעם אחת ו-Claude Code משתמש בהן שוב ושוב:

Skill לכתיבת טסטים:

# skill: write-tests ## description: כתיבת טסטים לפי הסטנדרט שלנו כשאתה כותב טסטים: 1. השתמש ב-Jest 2. כתוב לפחות 3 test cases: happy path, edge case, error case 3. השתמש ב-describe/it בעברית 4. הוסף mock לכל קריאת API חיצונית

Skill ל-Code Review:

# skill: code-review ## description: בדיקת קוד לפי הסטנדרטים שלנו בדוק: - אבטחה: SQL injection, XSS, הרשאות - ביצועים: שאילתות כבדות, לולאות מיותרות - קריאות: שמות משתנים, תיעוד, מבנה - טסטים: האם יש כיסוי מספיק?
💡

חלק ו׳: טיפים מתקדמים

עבודה מקבילית

הרץ כמה instances של Claude Code על חלקים שונים של הפרויקט בו-זמנית — כל אחד בטרמינל נפרד.

שילוב עם GitHub

> צור PR עם תיאור מפורט לשינויים שעשינו > בדוק את ה-PR של [שם עמית] ותן הערות

חיבור MCP Servers

הרחב את היכולות עם כלים חיצוניים — גישה לבסיסי נתונים, דפדפן, ועוד. MCP (Model Context Protocol) מאפשר ל-Claude Code להתחבר לשירותים נוספים.
